# WheelsParser

WheelsParser отслеживает публичные Telegram-каналы и чаты Twitch-стримов и находит новые ссылки вида `betboom.ru/freestream/...`. В Telegram-каналах ищутся ссылки и ключевые слова, в Twitch-чатах — только ссылки от стримера, модераторов, VIP и ботов. Найденные ссылки сохраняются локально и при наличии настроек отправляются в Telegram. Бот также отвечает на команды.

## Возможности

- мониторинг публичных каналов через `https://t.me/s/<channel>` без Telegram API для чтения;
- мониторинг Twitch-чатов через анонимное IRC-подключение (без токенов и OAuth): ловятся только ссылки `betboom.ru/freestream/...` от стримера, модераторов, VIP и известных ботов; ключевые слова в Twitch не ищутся (см. раздел «Twitch»);
- Telegram-уведомления через Bot API
- прямые запросы к API BetBoom для проверки статуса колёс;
- интерактивные команды бота, которые можно увидеть, прописав `/help`;
- управление списком каналов прямо из Telegram без правки `channels.txt`;
- управление списком ключевых слов прямо из Telegram без правки `keywords.txt`;
- единая канонизация URL колёс: query-параметры (utm и т.п.) и завершающий `/` отбрасываются везде — при извлечении ссылок, дедупликации, в `/active` и в кэше;
- дедупликация ссылок между каналами и сообщениями с кулдауном повторных уведомлений (`REALERT_COOLDOWN_MINUTES`);
- ручное удаление колеса из `/active` командой `/removewheel <номер>` — колёса в ответе `/active` нумеруются, колесо скрывается до конца суток (00:00 МСК), удаление переживает рестарт (`removed_wheels.json`); можно удалять и по ссылке;
- проверка статуса колеса через API BetBoom перед отправкой уведомления (`PRECHECK_WHEELS`) — завершившиеся колёса не рассылаются;
- все даты и счётчики (`/status`, `/wheels`, `/active`, `found_at`) считаются по московскому времени независимо от таймзоны сервера;
- offset Telegram-бота сохраняется в `bot_state.json` — после рестарта уже обработанные команды не выполняются повторно;
- сохранение истории в `freebets.json`;
- сохранение обработанных сообщений в `seen_ids.json`;
- первый запуск без уведомлений по старым сообщениям;
- «тихий» первый проход для каналов, добавленных через `/add` на лету;
- повторные сетевые запросы с backoff для `429` и `5xx`;
- атомарная запись JSON-файлов;
- ротация `parser.log`;
- цветной вывод и значки статусов в консоли;
- корректное завершение по `Ctrl+C`;
- настройка через переменные окружения.

## Команды бота

| Команда | Действие |
|---|---|
| `/start` | Приветствие и краткая информация о боте |
| `/active` | Проверка актуальных колёс за сегодня |
| `/removewheel <номер>` | Убрать колесо из `/active` по номеру из последнего ответа `/active` (скрывается до конца суток, 00:00 МСК). Работает и со ссылкой: `/removewheel <ссылка>`; алиас `/delwheel` |
| `/status` | Статистика найденных ссылок |
| `/wheels` | Колёса, найденные за последние 10 минут (окно настраивается через `WHEELS_WINDOW_MINUTES`); если ничего нет — сообщение об этом |
| `/channels` | Список отслеживаемых каналов |
| `/words` | Список ключевых слов |
| `/addword` | Добавить ключевое слово |
| `/removeword` | Удалить ключевое слово |
| `/add @channel` | Добавить канал в мониторинг (сохраняется в `channels.txt`) |
| `/remove @channel` | Убрать канал из мониторинга (сохраняется в `channels.txt`) |
| `/twitch` | Список отслеживаемых Twitch-каналов |
| `/addtwitch channel` | Добавить Twitch-канал (сохраняется в `twitch_channels.txt`; можно указать логин или ссылку `twitch.tv/логин`) |
| `/removetwitch channel` | Убрать Twitch-канал (сохраняется в `twitch_channels.txt`) |
| `/help` | Список команд и текущие настройки |

Меню команд регистрируется автоматически при запуске (`setMyCommands`), поэтому в Telegram появятся подсказки при вводе `/`.

## Файлы

- `betboom_web_parser.py` — основной скрипт;
- `requirements.txt` — зависимости Python;
- `run.bat` — установка и запуск в Windows;
- `.env.example` — шаблон настроек: скопируйте в `.env` и заполните;
- `.env` — локальные настройки и секреты (создайте сами, в архив не входит);
- `channels.txt` — список отслеживаемых каналов;
- `twitch_channels.txt` — список отслеживаемых Twitch-каналов (создаётся при первом добавлении канала);
- `keywords.txt` — список ключевых слов (создаётся автоматически);
- `freebets.json` — история уникальных ссылок (создаётся автоматически);
- `seen_ids.json` — состояние обработанных сообщений (создаётся автоматически);
- `bot_state.json` — сохранённый offset Telegram-бота (создаётся автоматически);
- `removed_wheels.json` — колёса, удалённые из `/active` командой `/removewheel` (создаётся автоматически);
- `parser.log` — создаётся автоматически при запуске.

## Быстрый запуск в Windows

1. Установите Python 3.10 или новее.
2. Скопируйте `.env.example` в `.env` и заполните `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
3. Запустите `run.bat`.

Скрипт сам установит зависимости и запустит парсер.

## Обычный запуск

```bash
python -m pip install -r requirements.txt
python betboom_web_parser.py
```

Ключи не следует записывать в код, README или публиковать в репозитории. Если одна из двух переменных не задана, парсер продолжит работать без уведомлений. Команды бота работают, если задан `TELEGRAM_BOT_TOKEN`.

## Настройки

| Переменная | По умолчанию | Назначение |
|---|---:|---|
| `TELEGRAM_BOT_TOKEN` | пусто | Токен Telegram-бота |
| `TELEGRAM_CHAT_ID` | пусто | Получатель уведомлений и единственный чат, где работают команды |
| `WHEELSPARSER_CHANNELS` | встроенный список | Каналы через запятую, с `@` или без. Используется **только при первом запуске**, если `channels.txt` ещё не существует: список сохраняется в файл. Дальше единственный источник правды — `channels.txt` (и команды `/add`, `/remove`), переменная игнорируется |
| `CHECK_INTERVAL` | `60` | Интервал проверки в секундах, минимум 10 |
| `REQUEST_TIMEOUT` | `15` | Тайм-аут HTTP-запроса |
| `MESSAGES_PER_CHANNEL` | `50` | Число последних сообщений для анализа |
| `MAX_SEEN_PER_CHANNEL` | `2000` | Максимум сохранённых ID на канал |
| `WHEELS_WINDOW_MINUTES` | `10` | Окно команды `/wheels` в минутах |
| `MAX_RESULTS` | `5000` | Максимум записей в `freebets.json`; при превышении старые записи отбрасываются |
| `ACTIVE_MAX_AGE_HOURS` | `20` | Максимальный возраст колеса для команды `/active`, часов |
| `ACTIVE_CHECK_CONCURRENCY` | `3` | Сколько потоков одновременно опрашивают API BetBoom при `/active`; у каждого потока своя HTTP-сессия |
| `REALERT_COOLDOWN_MINUTES` | `30` | Кулдаун повторного уведомления о том же URL, минут |
| `PRECHECK_WHEELS` | `true` | Проверять статус колеса через API BetBoom перед уведомлением; завершившиеся колёса пропускаются |
| `TWITCH_ENABLED` | `true` | Включить мониторинг Twitch-чатов |
| `WHEELSPARSER_TWITCH_CHANNELS` | пусто | Twitch-каналы через запятую. Используется **только при первом запуске**, если `twitch_channels.txt` ещё не существует; дальше источник правды — файл и команды `/addtwitch`, `/removetwitch` |
| `TWITCH_BOTS` | встроенный список | Логины чат-ботов через запятую, чьи ссылки принимаются без бейджа модератора (по умолчанию nightbot, streamelements, moobot, fossabot, wizebot, streamlabs) |
| `STALE_COMMAND_SECONDS` | `120` | Команды старше этого возраста подтверждаются, но не выполняются (защита от бэклога после простоя) |
| `CHANNEL_FAIL_THRESHOLD` | `5` | Уведомление о недоступном канале после N подряд неудачных циклов |
| `ALERT_ON_FIRST_RUN` | `false` | Отправлять находки из первого прохода |
| `USE_COLORS` | `true` | Цветной вывод в консоли |
| `USE_ICONS` | `true` | Эмодзи-значки в строках лога |

Если в классическом `cmd.exe` эмодзи отображаются квадратами, задайте `USE_ICONS=false` (будут текстовые метки `[OK]`, `[!]`, `[NEW]`) или запускайте парсер в Windows Terminal. В файл `parser.log` цветовые коды не записываются.

## Twitch

Парсер подключается к чатам Twitch анонимно через IRC (`irc.chat.twitch.tv:6697`, SSL) — регистрация приложения, токены и OAuth не нужны. Это штатный режим чтения чата (так работают все «анонимные» чат-вьюеры).

Как это работает:

- ссылка принимается, только если её отправил стример (бейдж broadcaster), модератор, VIP или известный бот из `TWITCH_BOTS`;
- найденные ссылки проходят ту же канонизацию URL, проверку статуса через API BetBoom (`PRECHECK_WHEELS`) и сохраняются в `freebets.json` с пометкой `"source": "twitch"`;
- кулдаун `REALERT_COOLDOWN_MINUTES` общий для всех источников: одно и то же колесо, найденное и в Telegram, и в Twitch, не рассылается дважды;
- уведомление из Twitch отправляется сразу, не дожидаясь очередного цикла проверки Telegram-каналов;
- при обрыве соединения парсер переподключается автоматически (backoff от 5 до 120 секунд); изменение списка каналов через `/addtwitch` и `/removetwitch` применяется на лету;
- важное отличие от Telegram: у IRC нет истории — читаются только сообщения, пришедшие пока парсер запущен. Ссылки, отправленные в чат во время простоя парсера, не восстанавливаются.

## Время

Все даты в проекте считаются по московскому времени (UTC+3, без летнего времени): метки `found_at` в `freebets.json`, счётчики `/status` «за сегодня», окно `/wheels` и суточный сброс `/active` в 00:00 МСК. Таймзона сервера на результаты не влияет.

## Формат данных

Каждая запись в `freebets.json` содержит URL, время обнаружения, канал, ID и URL исходного сообщения, фрагмент текста и результат отправки уведомления.
URL приводится к канонической форме: без query-параметров (utm и т.п.), без завершающего `/`, домен в нижнем регистре без `www`. Одна и та же ссылка с разными хвостами — это одна запись и одно уведомление.
Повторные находки одного и того же нормализованного URL (в любом канале) подавляются в течение `REALERT_COOLDOWN_MINUTES` (по умолчанию 30 минут). После кулдауна уведомление о том же адресе отправляется снова: колёса BetBoom живут на постоянных адресах, и один URL может означать новый запуск колеса. Если включён `PRECHECK_WHEELS`, статус колеса проверяется через API BetBoom до отправки, и уже завершившиеся колёса молча пропускаются.

`bot_state.json` хранит подтверждённый `update_id` Telegram-бота. Файл записывается атомарно после обработки каждой пачки команд; после рестарта парсера обработанные команды не выполняются повторно. При повреждении файла offset просто сбрасывается на 0 — защита от старого бэклога остаётся за `STALE_COMMAND_SECONDS`.
